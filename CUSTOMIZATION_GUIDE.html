How to Change Colors

1

Open styles.css

Navigate to the top of the file where variables are defined.

2

Find the :root section

:root {
	--primary-blue: #5ba3a8;
	--secondary-blue: #4a8b92;
	/* ... etc */
}
3

Replace the hex color code

For example, to change primary blue to a different shade:

--primary-blue: #YOUR_HEX_CODE_HERE;
4

Save and refresh your browser

The changes will automatically apply to all elements using that color variable.

üí° Tip: Use a color picker tool like HTML Color Codes or your browser's built-in inspector to find the perfect hex code.

Creating Color Gradients

The page uses gradient backgrounds in several places. Search for linear-gradient in styles.css to modify:

background: linear-gradient(
	135deg,
	var(--primary-blue) 0%,
	var(--secondary-blue) 100%
);

The angle (135deg), colors, and percentages can all be adjusted.

Changing Font Sizes

Find the element you want to modify and adjust the font-size property:

	font-size: 3.75rem;  /* Change this value */
	font-weight: 700;
}

Font Size Guidelines

  • Headlines (h1, h2): 2.5rem - 3.75rem
  • Subheadings (h3): 1.5rem - 2rem
  • Body text (p): 1rem - 1.2rem
  • Small text (labels): 0.75rem - 0.95rem

‚ú® Animation Customization

Understanding Animations

The page features sophisticated CSS animations defined in styles.css. All animations use @keyframes and are applied to elements via the animation property.

Main Animation Types

Animation Name Duration Applied To
float 12s Background shapes, organic motion
fadeInUp 0.8-1s Content elements on page load
fadeInScale 0.8s Hero visual card
slideInRight 1s Hero visual entrance
pulse-glow 3s (infinite) Visual card subtle glow effect

Modifying Animation Speed

Find the animation definition in styles.css:

animation: fadeInUp 0.8s cubic-bezier(0.34, 1.56, 0.64, 1) 0.1s both;
            /* Breakdown:
               - fadeInUp: animation name
               - 0.8s: duration
               - cubic-bezier: easing function
               - 0.1s: delay
               - both: fill mode */

Speed Adjustments:

  • Slower: Change 0.8s to 1.2s or 1.5s
  • Faster: Change 0.8s to 0.5s or 0.3s
  • Much slower: Change 0.8s to 2s for a more dramatic effect

Disabling Animations

To disable all animations (useful for accessibility), comment out or remove the animation property:

.hero-badge {
	/* animation: fadeInUp 0.8s cubic-bezier(0.34, 1.56, 0.64, 1) 0.1s both; */
}

Respecting User Preferences

For accessibility, the page should respect prefers-reduced-motion:

@media (prefers-reduced-motion: reduce) {
	* {
		animation: none !important;
		transition: none !important;
	}
}

Creating New Animations

To add a custom animation:

1

Define the keyframes

@keyframes myCustomAnimation {
	from {
		opacity: 0;
		transform: scale(0.8);
	}
	to {
		opacity: 1;
		transform: scale(1);
	}
}
2

Apply to an element

.my-element {
	animation: myCustomAnimation 0.8s ease-out forwards;
}
üí° Tip: Animations improve user engagement but can also distract. Use them purposefully to draw attention to important elements like CTAs and key features.

Button Styles

Modifying Button Styling

styles.css - Lines ~120-145
.cta-button.primary {
	background: linear-gradient(135deg, var(--primary-blue) 0%, var(--secondary-blue) 100%);
	color: var(--white);
	padding: 1rem 2rem;
	border-radius: 8px;
}

Customization Options:

  • Padding: Adjust button size with padding values
  • Border-radius: Change from 8px to 0px for sharp corners, or 50px for pill shape
  • Font size: Modify the font-size property
  • Hover effect: Modify the :hover state

Button Hover Effects

.cta-button.primary:hover {
	transform: translateY(-2px);
	box-shadow: 0 10px 25px rgba(91, 163, 168, 0.3);
}

Common Hover Effects:

  • transform: translateY(-2px) - Lifts button on hover
  • transform: scale(1.05) - Slightly enlarges button
  • box-shadow: 0 10px 25px - Adds shadow for depth
  • background: brightness(1.1) - Brightens the color

Adding Button Links

To make buttons functional, set the href or form action:

üì± Responsive Design Customization

Breakpoints

The page is optimized for three main screen sizes:

Breakpoint Screen Width Device Types
Desktop 1024px and above Desktops, large laptops
Tablet 768px - 1024px Tablets, small laptops
Mobile Below 768px Smartphones

Media Queries

Media queries are defined throughout styles.css:

/* Tablet */
@media (max-width: 1024px) {
	.hero-headline {
		font-size: 2.5rem;
	}
}

/* Mobile */
@media (max-width: 768px) {
	.hero-headline {
		font-size: 1.95rem;
	}
}

Adding Custom Breakpoints

/* New breakpoint for very small phones */
@media (max-width: 480px) {
	.hero-headline {
		font-size: 1.5rem;
	}
}

üìä Analytics & Tracking Setup

Google Analytics Integration

The page is ready for Google Analytics 4 (GA4) integration:

Step 1: Get Your Google Analytics ID

  • Go to Google Analytics
  • Create a new property for your landing page
  • Copy your Measurement ID (format: G-XXXXXXXXXX)

Step 2: Add GA4 to Your HTML

Add this to the <head> of your index.html:

<!-- Google Analytics -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-YOUR_ID"></script>
<script>
	window.dataLayer = window.dataLayer || [];
	function gtag(){dataLayer.push(arguments);}
	gtag('js', new Date());
	gtag('config', 'G-YOUR_ID');
</script>

Event Tracking

Track important user interactions:

Button Click Tracking

<button onclick="gtag('event', 'cta_click', {'button_location': 'hero'})">
	Get Started
</button>

Form Submission Tracking

<form onsubmit="gtag('event', 'signup_submit', {'form_location': 'contact'})">
	...
</form>

Video Play Tracking

<video onplay="gtag('event', 'video_play', {'video_id': 'video-1'})">
	...
</video>

Key Metrics to Track

  • Page Views: How many people visit the page
  • CTA Clicks: Clicks on "Get Started" and "Learn More" buttons
  • Form Submissions: Completed sign-up forms
  • Video Engagement: Video plays and watch time
  • Scroll Depth: How far down the page users scroll
  • Time on Page: How long users spend on the page
